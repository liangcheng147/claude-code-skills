# Common Clarification Dimensions

Domain-specific checklists for identifying which dimensions to clarify. Read this when triaging a request to find the right questions to ask.

## Table of Contents

- [Research and Information Organization](#research-and-information-organization)
- [Writing Tasks](#writing-tasks)
- [Data Analysis](#data-analysis)
- [Code or Product Changes](#code-or-product-changes)
- [Design or Visual Work](#design-or-visual-work)

---

## Research and Information Organization

- Time range
- Included/excluded categories
- Source authority
- Required fields
- Output format
- Completeness versus concise summary
- Citation/link requirements

## Writing Tasks

- Audience
- Purpose
- Tone
- Length
- Structure
- Source material
- Constraints or forbidden claims

## Data Analysis

- Decision or question to answer
- Input files and relevant columns
- Metrics
- Grouping and filtering
- Output format
- Whether charts are needed

## Code or Product Changes

- User-facing behavior
- Scope of files/modules
- Compatibility constraints
- Acceptance criteria
- Testing expectations
- Whether to preserve existing patterns

## Design or Visual Work

- Target audience
- Brand/style direction
- Functional requirements
- Platform and viewport
- Visual references
- Asset constraints
